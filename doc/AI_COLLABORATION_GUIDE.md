# AI_COLLABORATION_GUIDE.md

**Version:** 2.0

**Status:** Approved

**Architecture Level:** Enterprise

**Project:** Fateh Music Academy

---

# 1. Purpose

This document defines the official collaboration model between human developers and AI assistants during the development of the Fateh Music Academy platform.

The objective is to ensure that AI-generated code remains architecturally consistent, maintainable and predictable.

AI is considered an engineering assistant—not an architectural authority.

---

# 2. Vision

AI should increase

- Development Speed
- Code Quality
- Documentation Quality
- Consistency
- Productivity

without reducing architectural integrity.

---

# 3. Core Principle

AI may

Generate

Refactor

Explain

Optimize

Document

Review

code.

AI may NOT

change architecture without explicit approval.

AI may assist with

✓ Component Development

✓ Repository Implementation

✓ Utility Functions

✓ Documentation

✓ Unit Tests

✓ Refactoring

✓ Performance Improvements

✓ Accessibility

✓ SEO Content

Humans approve.

```
Related Documents
```

# 9. Coding Rules

AI-generated code must

✓ Follow coding standards

✓ Respect naming conventions

✓ Remain modular

✓ Be deterministic

✓ Avoid unnecessary abstraction

✓ Be readable

---

# 10. Documentation Rules

Every architectural change requires updates to

```
README

Architecture

Specifications

Folder Structure

API Documentation
```

Documentation is never optional.

---

# 11. Repository Rules

AI must never

Access page files directly.

Business data must be retrieved only through repositories.

---

# 12. Component Rules

Components must remain

Presentation-only.

AI should never move business logic into UI components.

---

# 13. SEO Rules

SEO metadata must only be produced through

```
SEO Engine
```

AI must never generate page-specific meta tags manually.

---

# 14. Schema Rules

JSON-LD must only be generated by

```
Schema Engine
```

Manual schema creation is prohibited.

---

# 15. Routing Rules

AI must never

Hardcode URLs.

Routes must always be generated by

```
Routing Engine
```

---

# 16. Image Rules

Images should always

Use Image Engine

Use responsive images

Contain alt text

Avoid hardcoded dimensions where unnecessary.

---

# 17. Content Rules

Content should

Remain structured

Follow topic clusters

Support internal linking

Remain AI-readable

---

# 18. Refactoring Rules

AI should recommend refactoring when

Responsibilities grow

Duplication appears

Complexity increases

Readability decreases

Behavior must remain unchanged.

---

# 19. Performance Rules

AI should prefer

Static Rendering

Minimal JavaScript

Small Components

Lazy Loading

Code Splitting

Performance regressions are unacceptable.

---

# 20. Accessibility Rules

Every generated component must

Support keyboard navigation

Use semantic HTML

Provide ARIA where appropriate

Meet WCAG 2.2 AA.

---

# 21. Security Rules

AI must never

Generate secrets

Hardcode credentials

Expose internal configuration

Bypass validation

---

# 22. Testing Rules

AI-generated features require

✓ Unit Tests

✓ Integration Tests

✓ Validation

✓ Documentation Updates
